    Functions to start and stop measurements

    Page 39 of GroPoint Profile User Manual says: A new measurement is triggered by a read request to the input registers (either read single or read multiple). The first read command will initiate the start of a measurement (soil moisture or soil temperature) and will respond to the master with an ACKNOWLEDGE (05) exception response. Moisture measurements take approximately 200 ms per segment. Temperature measurements take approximately 200 ms per sensor. The master should wait for this amount of time to expire before attempting to retrieve the measurement values with another read command.

    Functions to get one or more values from a sensor.

    Based off modbusMaster::getRegisters(), but with trigger followed by a 200 ms delay before read request

    Page 39 of GroPoint Profile User Manual says: A new measurement is triggered by a read request to the input registers (either read single or read multiple). The first read command will initiate the start of a measurement (soil moisture or soil temperature) and will respond to the master with an ACKNOWLEDGE (05) exception response. Moisture measurements take approximately 200 ms per segment. Temperature measurements take approximately 200 ms per sensor. The master should wait for this amount of time to expire before attempting to retrieve the measurement values with another read command.

    Page 37-38 of GroPoint Profile User Manual says: The input registers are read-only registers containing soil moisture or temperature measurement values. Moisture and temperature readings are stored as 16-bit signed integers and will be returned without a decimal place. In order to obtain the true result, the decimal place setting must be applied. The default setting for this is 1 decimal place, so for example a decimal value of 123 indicates 12.3 % volumetric moisture content for volumetric soil moisture measurements, or 12.3 C for the soil temperature sensor measurements.

    bool gropoint::begin(gropointModel model, byte modbusSlaveID, Stream* stream, int enablePin = -1)

    This function sets up the communication.

    Parameters
    model The model of the GroPoint sensor, from gropointModel
    modbusSlaveID The byte identifier of the modbus slave device.
    stream A pointer to the Arduino stream object to communicate with.
    enablePin A pin on the Arduino processor to use to send an enable signal to an RS485 to TTL adapter. Use a negative number if this does not apply. Optional with a default value of -1.
    Returns bool True if the starting communication was successful, false if not.

    It should be run during the arduino "setup" function. The "stream" device must be initialized prior to running this.

    byte gropoint::getSensorAddress(void)

    Gets the modbus sensor (slave) address.

    Returns byte The slave ID of the GroPoint sensor

    The address is in holding register 40201, decimal offset 200 (hexadecimal 0x00C8).

    Does not seem to work with a broadcast address of 0x00 or 0xFF

    bool gropoint::setSensorAddress(byte newSensorAddress)

    Set a new modbus sensor (slave) address.

    Parameters
    newSensorAddress The new address (slave ID) for the GroPoint sensor
    Returns bool True if the slave ID was successfully set, false if not.

    The address is in holding register 40201, decimal offset 200 (hexadecimal 0x00C8).

    The address change will take effect immediately, so any subsequent commands must use the new address.

    This register cannot be updated using the broadcast address.

    int16_t gropoint::getSensorBaud(void)

    Get the sensor modbus baud rate.

    Returns int16_t A code for the baud rate

    The baud rate is in holding register 40203, decimal offset 202 (hexadecimal 0x00CA)

    The factory default value is 0, corresponding to a baud rate of 19200.

    Valid values: 0=19200, 1=9600, 2=4800, 3=2400, 4=1200, 5=600, 6=300.

    bool gropoint::setSensorBaud(int32_t newSensorBaud)

    Set the sensor modbus baud.

    Parameters
    newSensorBaud The new baud rate to use
    Returns bool True if the baud rate was successfully set, false if not.

    The baud rate is in holding register 40203, decimal offset 202 (hexadecimal 0x00CA)

    Change this value to any of the following valid values: 19200, 9600, 4800, 2400, 1200, 600, 300.

    The new baud rate does not take effect until the sensor is power cycled, or if the restart communications diagnostic command (08, with subfunction 01) is received.

    String gropoint::getSensorParity(void)

    Get the Sensor modbus serial parity.

    Returns String The parity configuration

    The parity is in holding register 40204, decimal offset 0203 (hexadecimal 0x00CB).

    The factory default value is 2 corresponding to Even parity.

    bool gropoint::setSensorParity(String newSensorParity)

    Set the sensor modbus serial parity.

    Parameters
    newSensorParity the new parity, "None", "Odd", or "Even"
    Returns bool True if the parity was successfully set, false if not.

    The parity is in holding register 40204, decimal offset 0203 (hexadecimal 0x00CB).

    Change this value to any of the following valid values: "None", "Odd", "Even".

    The new parity setting does not take effect until the sensor is power cycled, or if the restart communications diagnostic command (08, with subfunction 01) is received.

    String gropoint::getSensorInfo(void)

    Returns String The sensor information in the format RIOTTECHGPLPTC vvvSNnnnnnn where vvv is the firmware version (v.v.v) and nnnnnn is the probe serial number.

    bool gropoint::restartCommunications(void)

    Restarts communication with the sensor using the modbus diagnostic command 08 (0x08) with subfunction 01.

    Returns bool True if the communication was restarted, false if not.

    A request data field contents of FF 00 hex causes the ports Communications Event Log to be cleared also. Contents of 00 00 leave the log as it was prior to the restart.

    bool gropoint::startMeasurement(void)

    Tells the sensors to begin taking measurements.

    Returns bool True if the measurements were successfully started, false if not.

    A start command is not in the GroPoint Modbus Manuals. However, Start/Stop functions are required to get these to work in ModularSensors. Instead, just confirm that sensor in communicating with a gropoint::getSensorAddress() command

    bool gropoint::stopMeasurement(void)

    Tells the sensors to stop taking measurements.

    Returns bool True if the measurements were successfully started, false if not.

    A stop command is not in the GroPoint Modbus Manuals; always returns true.
